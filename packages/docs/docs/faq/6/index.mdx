---
title: 「连接失败」可能是什么问题？
sidebar_position: 6
---

如果你尝试连接 Red 或 Satori 服务且失败了，以下几个小步骤可以迅速帮你定位问题。

首先，你需要使用「[如何判断 Chronocat 是否正在运行？](/faq/3)」中的方法，确定自己属于两类问题中的哪一类。

如果你看到了「恭喜」页面，那么你的问题可能是：

1. Chronocat 并未成功启动
2. QQ 尚未登录
3. 网络问题

否则，你的问题可能是：

4. 访问的路径问题
5. 服务密码（token）错误

## 1. Chronocat 并未成功启动

如果你还未检查过插件加载器是否安装成功，Chronocat 插件是否安装成功，那么应当首先检查。

在一台全新的电脑上，可以通过检查 [配置文件](/config) 是否生成来快速确定 Chronocat 是否成功启动。

## 2. QQ 尚未登录

登录后 Red 和 Satori 服务才会启动，因此请先登录 QQ。

## 3. 网络问题

网络问题是最常见的问题。特别注意，如果你的应用端和 Chronocat 不在同一台机器（Docker
也是如此），那么你需要进行组网。组网的方法请自行搜索。

## 4. 访问的路径问题

注意 Red 服务前的 `/api` 和 Satori 服务前的 `/v1`。

此外，Satori 事件服务的端点是 `/v1/events`。注意末尾的 `s`。

## 5. 服务密码（token）错误

服务密码错误或忘记填写服务密码也是常见的原因。

注意配置文件里的重写项。如果账号配置被重写，那么需要使用重写的服务密码。
